# 2105. Watering Plants II

This problem looks confusing because of the story. The trick is to ignore the story and convert it into a **two-pointer simulation**.

---

# Step 1: What is the question asking?

There are **n plants** arranged in a line.

```
0   1   2   3   4   5
```

Each plant needs some water.

Example

```
plants = [2,4,5,1,2]
```

means

```
Plant0 -> needs 2
Plant1 -> needs 4
Plant2 -> needs 5
...
```

There are two people.

* Alice starts from the **left**.
* Bob starts from the **right**.

Each has a watering can.

Initially

```
Alice water = capacityA
Bob water   = capacityB
```

Whenever someone reaches a plant

If current water is enough

```
Current water >= plant need
```

they simply water it.

Otherwise

```
Current water < plant need
```

they

1. refill
2. refill count++
3. water the plant

---

## Important

Refilling is **only when needed**.

Not after every plant.

# Step 3: Why two pointers?

Notice

Alice always moves

```
0 → 1 → 2 → 3
```

Bob always moves

```
n-1 → n-2 → ...
```

They never skip.

This immediately tells us

```
left pointer
right pointer
```

```
L                 R

0 1 2 3 4 5 6
```

After every step

```
L++
R--
```

This is exactly a two-pointer simulation.

# Final Algorithm

```
Initialize

AliceWater = capacityA
BobWater   = capacityB

left = 0
right = n-1

answer = 0

while(left < right)

    Alice waters left plant
        if water not enough
            refill++
            refill can
        water plant

    Bob waters right plant
        if water not enough
            refill++
            refill can
        water plant

    left++
    right--

If one plant remains

    Whoever has more water waters it
    If not enough
        refill++

Return answer
```

## Pattern to recognize

Whenever you see:

* One person starts from the **left**,
* Another starts from the **right**,
* Both move toward the center,
* Each position is processed once,

your first thought should be:

> **This is a two-pointer simulation problem.**

The main challenge isn't a complex algorithm—it's carefully simulating the rules (refill only when needed, and handling the single middle plant correctly).
```cpp
class Solution {
public:
    int minimumRefill(vector<int>& plants, int capacityA, int capacityB) {
        int left = 0, right = plants.size() - 1;
        int waterA = capacityA;
        int waterB = capacityB;
        int refill = 0;

        while (left < right) {
            // Alice waters from the left
            if (waterA < plants[left]) {
                refill++;
                waterA = capacityA;
            }
            waterA -= plants[left];

            // Bob waters from the right
            if (waterB < plants[right]) {
                refill++;
                waterB = capacityB;
            }
            waterB -= plants[right];

            left++;
            right--;
        }

        // If one plant remains (both meet at the same plant)
        if (left == right) {
            if (max(waterA, waterB) < plants[left]) {
                refill++;
            }
        }

        return refill;
    }
};
```
