## LeetCode 169. Majority Element

### Problem Statement (Short)

You are given an integer array `nums` of size `n`.

Find the **majority element**, i.e., the element that appears **more than ⌊n/2⌋ times** in the array.

It is **guaranteed** that a majority element always exists.

### Example

```text
Input:  nums = [2,2,1,1,1,2,2]

n = 7

⌊7/2⌋ = 3

Element 2 appears 4 times (>3)

Output = 2
```

---

## Constraints

```text
1 <= nums.length <= 5 * 10^4

-10^9 <= nums[i] <= 10^9

A majority element is always present.
```

### What do the constraints tell us?

* `n` can be as large as **50,000**, so an `O(n²)` solution is too slow.
* Array values can be very large or negative, so don't use the values as array indices.
* Since a majority element is guaranteed, we don't need to check whether it exists after finding it.

---

# Optimal Approach (Boyer-Moore Voting Algorithm)

### Idea

Keep two variables:

* `candidate` → current majority candidate.
* `count` → vote count of the candidate.

Rules:

1. If `count == 0`, choose the current element as the new candidate.
2. If the current element equals the candidate, increase `count`.
3. Otherwise, decrease `count`.

Since the majority element appears more than all other elements combined, it can never be completely canceled out and will remain as the final candidate.

---

## Dry Run

```text
nums = [2,2,1,1,1,2,2]
```

| Element | Candidate | Count |
| ------: | :-------: | ----: |
|       2 |     2     |     1 |
|       2 |     2     |     2 |
|       1 |     2     |     1 |
|       1 |     2     |     0 |
|       1 |     1     |     1 |
|       2 |     1     |     0 |
|       2 |     2     |     1 |

Final Answer = **2**

---

## C++ Code

```cpp
class Solution {
public:
    int majorityElement(vector<int>& nums) {

        int candidate = 0;
        int count = 0;

        for (int num : nums) {

            if (count == 0)
                candidate = num;

            if (num == candidate)
                count++;
            else
                count--;
        }

        return candidate;
    }
};
```

---

## Time & Space Complexity

* **Time Complexity:** `O(n)` (single traversal)
* **Space Complexity:** `O(1)` (only two variables)

### Interview Tip

If the interviewer **does not guarantee** that a majority element exists, after finding the `candidate`, make one more pass through the array to count its occurrences and verify that it appears more than `n/2` times.
